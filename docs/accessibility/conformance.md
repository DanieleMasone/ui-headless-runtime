# Accessibility statement

The project targets the interaction and semantic requirements associated with WCAG 2.2 AA and WAI-ARIA Authoring Practices patterns. Automated axe checks, real-browser keyboard tests, focus assertions, relationship checks, and manual review guidance are part of the repository.

This is not a claim that every consumer result automatically conforms. Consumers control markup, labels, content, contrast, zoom/reflow, announcements, target size, error language, and assistive-technology validation. The demo documents these boundaries on every component page.

Report accessibility defects through the GitHub issue tracker with browser, assistive technology, input method, reduced-motion/contrast settings, and a minimal reproduction.
