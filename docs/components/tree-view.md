# Tree View

## Overview and use cases

Tree View navigates hierarchical entities with expansion and optional single/multiple selection. Use nested lists for non-interactive document outlines.

## Options, snapshot, and registration

Expansion and selection have independent controlled/uncontrolled options. Register parent IDs, has-children metadata, and asynchronous `loading` state dynamically. Visible snapshots compute level, set size, position, expanded, selected, disabled, loading, and roving tabindex.

## Commands, events, keyboard, and focus

Toggle, select, set active, and forward keydown. beforeExpand and beforeSelect are cancellable. Up/Down traverse visible nodes; Right expands or enters a child; Left collapses or reaches the parent; Home/End and typeahead are supported.

## Async/dynamic edges and cleanup

Nodes may declare children before async registration and expose loading through `aria-busy` in the consumer renderer. Removing the active node chooses a valid visible neighbor. Traversal protects against cycles. Release registrations and destroy external subscriptions/timers.

The live nested, disabled, and dynamic-loading scenarios are generated by the shared [`createExample()` source](../../apps/demo/src/examples/create-example.ts), not by documentation-only mocks.
